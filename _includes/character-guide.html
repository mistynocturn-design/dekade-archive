Characters editor guide

기본 사용 순서

  1. Characters 화면의 + 버튼을 눌러 Character Manager를 엽니다.
  2. Character file에 이름, 이미지, 표시 순서와 색상을 입력합니다.
  3. Add content에서 원하는 항목을 작성하고 각 Add 버튼을 눌러 추가합니다.
  4. Current items에서 추가된 항목을 수정하거나 순서를 옮기고, 오른쪽 Preview로 결과를 확인합니다.
  5. Copy Markdown을 누른 뒤 결과물을 _characters/파일명.md에 붙여넣고 발행합니다. Download .md로 파일을 받아 넣어도 됩니다.
  6. 이미지는 assets/characters/캐릭터명/ 안에 넣고, 에디터에는 /dekade-archive/assets/characters/캐릭터명/파일명.jpg 형식으로 적습니다.

Character file

Character name
카드와 상세 페이지에 표시되는 캐릭터 이름입니다.
File name
완성 파일의 이름입니다. 자동 생성된 값은 그대로 사용해도 됩니다.
Card order
Characters 첫 화면의 카드 순서입니다. 숫자가 작을수록 먼저 표시됩니다.
Subtitle
상세 페이지에서 캐릭터 이름 아래에 표시되는 짧은 설명입니다.
Small role text
Characters 카드에서 이름 위에 작게 표시되는 역할이나 직업입니다.
Card image
Characters 카드 목록에 사용하는 대표 이미지입니다.
Profile image
상세 Profile 화면의 왼쪽 카드에 사용하는 이미지입니다.
Accent
캐릭터의 주요 강조색입니다.
Soft
연한 보조색과 일부 배경에 사용하는 색입니다.
Ink
본문과 주요 글자에 사용하는 기본 글자색입니다.
Tab color
Profile, Status, Story, AU, Memo 메뉴의 선택 글자와 움직이는 밑줄 색입니다.
Section text
NAME, PROFILE 같은 Section title의 글자색입니다.
Keyword bg
키워드의 작은 사각형 배경색입니다.

Add content

Card facts
성별, 나이, 키처럼 대표 카드에 넣을 짧은 정보입니다. 처음 3개는 Characters 카드에도 보이고, 전체 항목은 상세 Profile의 왼쪽 카드에 표시됩니다.
Profile
상세 Profile 오른쪽 정보 박스입니다. Section은 큰 분류, Small label은 항목명, Text는 내용, Small note는 덜 강조되는 보충 설명입니다. Note placement로 같은 줄이나 다음 줄을 고릅니다.
Keywords
성격이나 특징을 짧은 키워드 상자로 추가합니다. 같은 Section 이름을 사용하면 한 묶음으로 모입니다.
Details
Profile 아래의 큰 상세 박스입니다. Section title로 큰 구획을 만들고, Toggle title로 접었다 펼 수 있는 항목을 추가합니다. 본문과 여러 이미지, 처음부터 열어둘지 여부를 함께 지정할 수 있습니다.
Status
능력명과 0~100 수치를 막대그래프로 표시합니다.
Story
사건과 서사를 타임라인으로 정리합니다. Date는 비워도 되고, 정확한 날짜 대신 유년기나 성인기 같은 글자를 적어도 됩니다.
AU
다른 세계관의 캐릭터 설정을 카드 갤러리와 상세 Profile 형식으로 추가합니다. 여러 줄 입력란은 안내된 | 구분 형식을 사용합니다.
Memo
날짜와 짧은 제목을 곁들인 자유 메모입니다. 날짜와 제목은 필요 없으면 비워도 됩니다.

편집과 저장

Current items에서는 탭별로 항목을 골라 수정, 위아래 이동, 삭제할 수 있습니다. 항목을 수정한 뒤에는 Update 버튼을 눌러 반영합니다.

Save draft는 현재 작업을 이 브라우저에 임시 저장하고, Load selected는 선택한 초안을 불러옵니다. 캐릭터마다 다른 Draft name을 쓰면 여러 명을 따로 보관할 수 있습니다. Draft는 GitHub에 올라가는 파일이 아니므로 최종본은 반드시 Copy Markdown 또는 Download .md로 저장해야 합니다.

Refresh output은 현재 입력으로 결과 코드를 다시 만들고, Copy Markdown은 그 코드를 복사합니다. Preview는 지금 편집 중인 탭의 실제 화면을 미리 보여줍니다.
